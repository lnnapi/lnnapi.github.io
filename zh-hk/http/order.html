# 訂單模塊(SpotOrder)

# 訂單查詢(QueryOrder)

接口: /SpotOrder/QueryOrder
請求方式: GET
參數列表:

參數名類型參數説明
SideInt交易方向, 0:全部 1:買入 2:賣出
TradeSymbolString交易對名稱, 例如: BTC_USDT
OrderStateInt32訂單狀態 1:待成交 2:已成交 3:已取消
StartTimeInt64查詢訂單開始時間戳精確到秒, 0代表不限製 [選填參數默認值0]
EndTimeInt64查詢訂單結束時間戳精確到秒, 0代表不限製 [選填參數默認值0]
PageIndexInt32查詢第幾頁(每頁20條記錄), 從1開始 [選填參數默認值1]

返回值:

{
    "OrderList":
	[
		{
			"OrderId": 訂單Id(String),
			"Side": 交易方向, 0:全部 1:買入 2:賣出(Int),
			"TradeSymbol": 交易對名稱(String),
			"Price": 訂單價格(Float64),
			"Amount"： 訂單的交易數量(Float64),
			"AvgPrice": 已成交平均價格(Float64),
			"FinishAmount": 已成交數量(Float64),
			"TradeTotalSum": 已成交金額(Float64),
			"TotalSum": 訂單的委托金額(Float64),
			"OrderType": 訂單類型(Int),
			"Fee": 手續費(Float64),
			"StopLimitType": 計劃委托觸發類型(Int),
			"StopLimitPrice": 計劃委托的觸發價格，僅計劃委托單才有此數據(Float64),
			"CreateTime": 訂單創建時間精確到毫秒(Int64),
			"State": 訂單狀態(Int)
		},
		...
	],
	"DataTotalNum": 滿足條件的總數量(Int32)
}

備註:

# 訂單類型

取值說明
1普通限價單
2市價單
3IOC
4FOK
5計劃委托
6PostOnly限價單

# 訂單狀態

取值說明
1訂單已創建，但未進入撮合隊列
2計劃委托單等待觸發中
3撮合中
4訂單已完成
5訂單由於內部錯誤而取消
6用戶手動取消中
7用戶已手動取消
8訂單已被系統取消

# 計劃委托觸發類型

取值說明
1大於等於觸發價格時觸發
2小於等於觸發價格時觸發

# 計劃委托觸發後訂單執行類型

取值說明
0限價執行
1市價執行

# 創建計劃委托單(CreateStopLimitOrder)

接口: /SpotOrder/CreateStopLimitOrder
請求方式: POST
參數列表:

參數名類型參數説明
TradeSymbolString交易對名稱, 例如: BTC_USDT
QuoteFloat普通限價單為價格; 市價買為最大成交額; 市價賣為0
BaseFloat普通限價單為數量; 市價買為0; 市價賣為最大賣出數量
StopLimitTypeInt觸發類型
StopLimitPriceFloat觸發價格
SideInt交易方向, 1:買入 2:賣出
OrderTypeInt32觸發後訂單執行類型

返回值:

{
	"OrderId"：訂單Id
}

# 創建市價單(CreateMarketPriceOrder)

接口: /SpotOrder/CreateMarketPriceOrder
請求方式: POST
參數列表:

參數名類型參數説明
TradeSymbolString交易對名稱, 例如: BTC_USDT
SideInt交易方向, 1:買入 2:賣出
AmountFloat最大成交數量[市價買為選填]
TotalSumFloat最大成交額[市價賣為選填]

返回值:

{
	"OrderId"：訂單Id
}

特殊說明

  • 市價賣單可以只填TotalSum, 系統會凍結所有可用Base幣種，並以不超過TotalSum的交易額為目標盡可能執行
  • 市價買單可以只填Amount, 系統會凍結所有可用Quote幣種，並以不超過Amount的數量為目標盡可能執行

# 批量創建訂單(BatchedCreateOrders)[僅限高級用戶使用]

接口: /SpotOrder/BatchedCreateOrders
請求方式: POST
參數列表:

參數名類型參數説明
TradeSymbolString交易對名稱, 例如: BTC_USDT
OrderTypeInt訂單類型[僅限普通限價單和PostOnly限價單]
OrderDataString批量訂單參數組合數據，訂單的不同參數以英文逗號分隔，不同訂單信息使用英文豎線分隔[批量創建訂單所有參數都為必填參數]

返回值:

{
	"SuccessList"：創建成功的訂單Id列表
	[
		Order Id(String),
		...
	]
}

備註:

  • 批量創建訂單時，一次最多100個訂單並依次處理，如果遇到任意訂單執行失敗，則立即停止處理
  • 批量創建限價訂單格式為: {Side},{Amount},{Price}|{Side},{Amount},{Price}

TradeSymbolString交易對名稱, 例如: BTC_USDT
OrderIdString訂單Id

返回值: 無

# 批量取消訂單(BatchedCancelOrders)[僅限高級用戶使用]

接口: /SpotOrder/BatchedCancelOrders
請求方式: POST
參數列表:

參數名類型參數説明
TradeSymbolString交易對名稱, 例如: BTC_USDT
OrderIdString訂單Id列表, 使用英文豎線

返回值:

{
	"SuccessCount": 成功處理個數,
	"FailList": 失敗的訂單Id列表
	[
		Order Id(String),
		...
	]
}

備註:

  • 批量取消訂單將依次處理所有取消訂單請求, 遇到執行失敗, 會跳過繼續處理

返回值:

{

}

  • 取消所有訂單，但每次最多取消100條訂單
