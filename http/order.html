# SpotOrder

# Query Order

Interface: /SpotOrder/QueryOrder
Method: GET
Parameter:

ParameterTypeExplain
SideIntTrade side, 0:All 1:Buy 2:Sell
TradeSymbolStringTrade symbol name, eg, BTC_USDT
OrderStateInt32Order state 1:Open Order 2:Executed 3:Canceled
StartTimeInt64Start time for query, accurate to second, zero means no limit [default value: 0]
EndTimeInt64End time for query, accurate to second, zero means no limit [default value: 0]
PageIndexInt32Query page(20 items each page) starts from 1 [default value: 1]

Response:

{
    "OrderList":
	[
		{
			"OrderId": Order Id (String),
			"Side": Trade side, 0:All 1:Buy 2:Sell (Int),
			"TradeSymbol": Trade symbol name (String),
			"Price": Order price (Float64),
			"Amount"： Order amount (Float64),
			"AvgPrice": Average trade price (Float64),
			"FinishAmount": Executed amount (Float64),
			"TradeTotalSum": Excuted volume (Float64),
			"TotalSum": Order total volume (Float64),
			"OrderType": Order type (Int),
			"Fee": Fee (Float64),
			"StopLimitType": Stoplimit trigger condtion (Int),
			"StopLimitPrice": Stoplimit trigger price (Float64),
			"CreateTime": Order Creation timestamp accurate to millisecond (Int64),
			"State": Order state(Int)
		},
		...
	],
	"DataTotalNum": Total count mathcs the condtion (Int32)
}

Remarks:

# OrderType

ValueExplain
1Limit
2Maret
3IOC
4FOK
5StopLimit
6PostOnly

# Order State

ValueExplain
1Created, waiting for match
2StopLimit order waiting for trigger
3Matching
4Fully executed
5Canceld bacause of inner error
6Canceling by user
7Canceled by user
8Canceled by system

# StopLimit Order Trigger Condition

ValueExplain
1Greater or equal to trigger price
2Less or equal to trigger price

# Executed Order Type whe Triggered

ValueExplain
0Executed by limit order
1Executed by market order

# Create Market Pric eOrder

Interface: /SpotOrder/CreateMarketPriceOrder
Method: POST
Parameter:

ParameterTypeExplain
TradeSymbolStringTrade symbol name, eg, BTC_USDT
SideIntTrade side, 0:All 1:Buy 2:Sell
AmountFloatMaximum executed amount[Optional parameter if it's market buy]
TotalSumFloatMaximum executed volume[Optional parameter if it's market sell]

Response:

{
	"OrderId": Order Id (String)
}

Remarks:

  • If Market sell only sets "TotalSum", the system will freeze all base coin and trade as much as possible but will not exceed 'TotalSum'
  • If Market buy only sets "Amount", the system will freeze all quote coin and trade as much as possible but will not exceed 'Amount'

# Batched Create Orders[VIP only]

Interface: /SpotOrder/BatchedCreateOrders
Method: POST
Parameter:

ParameterTypeExplain
TradeSymbolStringTrade symbol name, eg, BTC_USDT
OrderTypeIntOrders type[Can only be set to Limit or PostOnly]
OrderDataStringCombined data, each order's parameters separated by comma, different orders separated by Y-axis[All parameters for each order must be set]

Response:

{
	"SuccessList"： Successul Order Id List
	[
		Order Id (String),
		...
	]
}

Remarks:

  • Each batched call can create at most 100 orders, which are processed one by one. The procedure will be stopped if one order is failed to execute.
  • Batched create limit order: {Side},{Amount},{Price}|{Side},{Amount},{Price}

# Batched Cancel Orders[VIP only]

Interface: /SpotOrder/BatchedCancelOrders
Method: POST
Parameter:

ParameterTypeExplain
TradeSymbolStringTrade symbol name, eg, BTC_USDT
OrderIdStringOrder Id list, at most 100

Response:

{
	"SuccessCount": Successful count,
	"FailList": Failed Order Id List
	[
		Order Id(String),
		...
	]
}

Remarks:

  • Each batched call can cancel at most 100 orders, which are processed one by one. The procedure will continue and ignore the failed order.
